Mission Control Slice 1.1 Notes

Summary

Status: complete · Date: 2026-06-16 · Task: mobile shell polish and Revenue lane.

  • Mobile nav now behaves like centered app chrome, with safe-area padding and a capped rail width.
  • Revenue now routes attention closest to consulting/job/app cash decisions with live local data.
  • Work now exposes explicit Todo/Doing/Done controls on mobile task rows instead of hiding status changes behind the colored dot.
  • Work now sorts tasks by priority first and uses visible priority color treatment.
  • Work task inspection now has real actions: ranking explanation, status controls, priority controls, defer, and archive.
  • Command now has a real attention brief instead of the static Since Last Look placeholder.
  • No backend/schema rewrite for this slice; use current local APIs/files and client adapters.

Design Decisions

  • Keep five mobile nav items visible: Command, Work, Revenue, Ship, Machine.
  • Tailwind must scan lib/ because Mission Control now keeps shared class constants there.
  • Move legacy consulting strategy content out of the primary Revenue lane rather than deleting it.
  • Status changes in the redesigned Work lane must be visible controls, not implied dot interactions.
  • Newest-updated order is not a good Work default; priority and active/problem states come before recency.
  • The drawer should explain why a task is ranked and let JT act without leaving the Work lane.
  • The Command homepage should compress live attention state: top action, latest proof, urgent JT count, revenue pressure, and risk count.
  • Ship should be a real /ship lane, not the old /vibe page with a new nav label.
  • Machine should be a real /machine lane, not the old /agents page with a new nav label.

Deviations

Kept the old consulting strategy page under /legacy/consulting so the primary /consulting lane can become the cash-path cockpit without deleting prior work.

Kept a lightweight legacy Vibe reference under /legacy/vibe; /vibe now redirects to /ship so stale links cannot show the old marketing-system page as the active Ship lane.

Kept the old Agent Team screen under /legacy/agents; /agents now redirects to /machine so stale links cannot show the agent directory as the active Machine lane.

Kept the old Audit Trail screen under /legacy/audit; /audit now redirects to /evidence so stale links cannot show the old proof log as the active Evidence lane.

Kept the old Deployment Monitor and Cost Dashboard under /legacy/monitor and /legacy/costs; /monitor and /costs now redirect to /health so stale links cannot show placeholders as the active Health lane.

Tradeoffs

  • Static/client aggregation is faster and safer for Slice 1.1; a shared /api/signals aggregator remains deferred.

Open Questions

None blocking.
