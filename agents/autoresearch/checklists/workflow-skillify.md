# workflow-skillify Autoresearch Checklist

Answer yes/no for each evaluated run:
1. Did it identify the concrete trigger phrases/events for the reusable workflow?
2. Did it choose the smallest durable artifact: checklist, skill, script, or regression check?
3. Did it capture edge cases, “do not” rules, quality gates, and done state?
4. Did it check existing skills before creating a new one?
5. Did it add regression checks when the source was a correction/failure?
6. Did it update daily note and any required registration/autoresearch surfaces?
