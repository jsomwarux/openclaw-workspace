# Autoresearch Checklist — client-proof-capture

1. Does it block public proof until acceptance, evidence, permission/anonymization, and sensitivity gates are explicit?
2. Does it route active clients through Client OS files?
3. Does it capture edge cases, failures, judgment calls, and reusable IP?
4. Does it distinguish blocked, private-ready, and publish-ready proof states?
5. Does it prevent unverified ROI, client acceptance, or public naming claims?
6. Does it produce concrete next actions and file paths?
