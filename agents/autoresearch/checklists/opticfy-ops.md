# Autoresearch Checklist — opticfy-ops

Score client-ops outputs against these yes/no checks:

1. Does the output capture the client's real workflow before suggesting automation?
2. Does it turn client-specific work into reusable IP, templates, or runbooks?
3. Does it include edge cases, failure modes, and judgment calls?
4. Does it distinguish current-state documentation from future-state recommendations?
5. Does it produce a client-ready artifact, not just internal notes?
6. Does it avoid automating unclear or unverified process steps?
