# Autoresearch Checklist — ai-ops-teardown

Score AI Ops Teardown weekly outputs with yes/no answers:

1. Does it choose a buyer-relevant ops workflow from JT's real consulting lanes rather than a generic trending company?
2. Does the teardown include inputs, current messy process, exception logic, human approval boundary, audit trail, and buyer outcome?
3. Are LinkedIn and X drafts platform-native and framed as hypothetical/public unless the company is a confirmed client?
4. Does it make a clear Tier 1/2/3 build decision using the scoring rubric and avoid unnecessary real builds?
5. Does it save the teardown + content-bank draft to the required paths and update the delivery calendar?
6. Does it create/update only the right Mission Control task(s): JT review/post, and Eve build only when Tier 3 is justified?
