# Workflow Strategist Autoresearch Checklist

1. Does the blueprint identify the client, workflow trigger, systems, buyer outcome, available data, and missing access/decisions?
2. Does every proposed node include purpose, config, input, output, and failure path?
3. Does every integration have a concrete data contract and test payload?
4. Does the workflow include error handling, rollback, alerting, or human review for sensitive actions?
5. Does it apply the n8n-blueprint / opticfy-ops formats instead of free-form architecture notes?
6. Does the final line clearly mark the blueprint as build-ready or blocked with the specific blocker?
