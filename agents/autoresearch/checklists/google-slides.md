# Autoresearch Checklist — google-slides

Score deck generation workflows against these yes/no checks:

1. Does the deck have a clear buyer-facing narrative and concrete problem framing?
2. Does every slide have one job, not a wall of mixed ideas?
3. Does it use JT's real proof points, niche context, or client-relevant examples?
4. Does it avoid generic AI consultant language and vague transformation claims?
5. Does it create/export/upload the deck successfully and return the Drive link?
6. Does it include a review-ready next step for JT?
