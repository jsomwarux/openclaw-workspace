# Runbook Skill Checklist
> Evaluates whether Eve's generated runbooks are accurate, current, and actionable.
> Run on a sample of runbooks generated in the past 7 days.

**Scoring:** fraction of items passing. Target: 0.90.

---

## Checklist

1. **n8n node names are current and accurate.** Does the runbook reference correct n8n node names (e.g., "WhatsApp Trigger", "Telegram Trigger", "HTTP Request", "Claude Node")? Check against current n8n documentation if uncertain. Outdated node names = fail.

2. **Webhook URLs are current.** Do the webhook trigger URLs in the runbook match the current deployment? If the runbook was generated for a specific workflow, verify the webhook endpoint is still valid.

3. **Escalation path matches actual decision tree.** Does the escalation logic in the runbook match how decisions actually flow in the described process? Runbooks that invent escalation rules that don't exist = fail.

4. **Tool versions referenced are accurate.** If the runbook mentions specific tools (n8n version, Claude model, specific API versions), are those still current? A runbook that references n8n v0.180 when current is v1.x = fail.

5. **Steps are in the correct order.** Does the runbook's sequence of steps match the actual operational sequence? Misordered steps that would confuse an operator = fail.

6. **Actionable exit conditions specified.** Does the runbook specify what to do when the process fails at each step? A runbook that describes the happy path but not the failure path = partial fail (count as 0.5).

---

**Run method:** Eve reads the generated runbook file and checks each item.
**Test inputs:** Last 3 runbooks generated by the runbook skill.
