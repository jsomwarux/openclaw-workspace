# Runbook Skill Checklist
> Evaluates whether Eve's generated runbooks are accurate, current, and actionable.
> Run on a sample of runbooks generated in the past 7 days.

**Scoring:** fraction of items passing. Target: 0.90.

---

## Checklist

1. Does the runbook reference current, accurate tool/workflow names rather than stale labels?
2. Does it verify webhook URLs, file paths, cron IDs, or deployment endpoints against the current workspace before presenting them as operational facts?
3. Does the escalation path match the actual decision tree instead of inventing approval or handoff rules?
4. Are tool versions, model names, API names, and CLI commands current or explicitly marked as assumptions when unverifiable?
5. Are the operational steps in the correct execution order with prerequisites before dependent actions?
6. Does each failure point include an actionable exit condition or recovery step, not just the happy path?

---

**Run method:** Eve reads the generated runbook file and checks each item.
**Test inputs:** Last 3 runbooks generated by the runbook skill.
