# Autoresearch Checklist — client-proof-engine

1. Does the agent load active client status, proof gates, Client OS, and client folders first?
2. Does it keep proof private by default unless permission/anonymization is clear?
3. Does it inventory evidence paths instead of making unsupported claims?
4. Does it route outputs to proof, IP, portfolio, content, referral, or Mission Control only when gates allow?
5. Does it create Mission Control tasks only with concrete first action, why, and done state?
6. Does it avoid outreach sends and direct publishing?
