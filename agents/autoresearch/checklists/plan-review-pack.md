# Autoresearch Checklist — plan-review-pack

Score each item yes/no against a produced review pack.

1. Does the pack name the objective, reviewer, source plan/spec, and related task or file?
2. Does it separate scope/non-goals/assumptions from the proposed plan?
3. Does it ask for exact human signal on direction, risk, missing edge cases, acceptance, or good-enough-to-ship?
4. Does it include acceptance criteria and proof/demo target or an explicit no-proof-needed reason?
5. Does it name privacy/security/external-action/finance/auth-model-config boundaries when relevant?
6. Does it route feedback back to a source artifact, Client OS, Mission Control task, or implementation notes with owner/date?
