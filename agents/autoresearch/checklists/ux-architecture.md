# Autoresearch Checklist — ux-architecture

Score UX architecture outputs against these yes/no checks:

1. Does the recommendation start from the user's job-to-be-done rather than visual preference?
2. Does it define the primary screen/action hierarchy clearly?
3. Does it reduce cognitive load instead of adding decorative complexity?
4. Does it include edge/error/empty states where relevant?
5. Does it produce implementation-ready structure, copy, or acceptance criteria?
6. Does it avoid generic UI advice that could apply to any product?
