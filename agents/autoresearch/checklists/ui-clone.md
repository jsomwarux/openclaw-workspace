# UI Clone Autoresearch Checklist

1. Does the output extract and reuse visible colors, spacing, typography, radius, and layout hierarchy from the screenshot instead of inventing a new style?
2. Does it preserve every visible major element from the screenshot without adding unrelated UI?
3. Does it choose the correct implementation target for the project (Next.js/Tailwind for web, SwiftUI for iOS) and integrate with the local design system when required?
4. Does it include responsive sizing so text, controls, and fixed-format surfaces do not overlap or shift awkwardly?
5. Does it add only expected interaction states such as pressed/scroll behavior, without changing the screenshot's visual intent?
6. Does the final answer produce a complete renderable screen/component rather than a design description?
