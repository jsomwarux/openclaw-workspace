# /anomaly-audit

Packaged quick-win engagement. Client provides a data export (CSV/Excel); Eve runs structured
AI analysis to surface revenue leaks, pricing errors, and operational anomalies.
Deliverable: executive summary + anomaly register + estimated revenue impact.

Positioned as: *"You likely have 3-7% revenue sitting in data you already have. We find it."*

## Usage

```
/anomaly-audit $ARGUMENTS
```

Client provides data file. Run analysis type-specific checks below, then generate report.

## Analysis by Data Type

### Pricing / Margin (wholesale distributors)
- SKUs priced below cost (cost > price)
- Margins below niche floor (< [X]% for commodity, < [X]% for specialty)
- Price inconsistency: same SKU sold at different prices to different customers
- Discount abuse: discounts applied outside policy thresholds
- Inactive SKUs still generating orders (fulfillment risk)
- Price stale dates: last price update > 12 months on high-velocity items

### Rent Roll (property managers)
- Units below market rate (compare to StreetEasy / Zillow comparables)
- Vacancy clustering: multiple vacancies in same building/floor = structural issue
- Lease expiry concentration: >30% expiring same quarter = renewal risk
- Expense anomalies: utilities/maintenance significantly above or below comparable units
- Late fee patterns: same tenants repeatedly late = collection risk
- Concession analysis: are free months eating into effective yield?

### Claims (insurance operations)
- Claims above avg for same policy type or zip code
- High-frequency claimants (same policyholder multiple claims in 12 months)
- Billing inconsistencies: same procedure coded differently across claims
- Approval rate outliers: adjuster approval rates significantly above/below average
- Cycle time anomalies: claims open significantly longer than avg without status change
- Overpaid premiums: coverage gaps vs. premium charged

### Inventory (general operations)
- Dead stock: items with no movement in 90+ days
- Reorder point mismatches: items below reorder point not flagged
- Supplier price drift: same item from same supplier with price variance >10%
- Shrinkage outliers: location-level variance above acceptable threshold

## Tips

1. **Quantify everything.** Approximate dollar impact beats precise "this looks off."
2. **Show your work.** Reference the specific rows/items for each finding. Client needs to verify.
3. **Acknowledge limits.** If data is incomplete, say so — protects credibility.
4. **Always end with the upsell.** "This pattern recurs — we can automate monthly detection for $X/month."
