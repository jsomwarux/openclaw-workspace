# Outreach Rules — Full Detail
> Referenced from AGENTS.md. Read this file when handling outreach tracking.

## Outreach Status Tracking Rule (mandatory)
When JT says he sent an outreach message, connection request, or follow-up to ANY prospect — do ALL of the following in the same turn, immediately. Do NOT defer.

## Warm-Up Comment Strategy Rule
Warm-up comments are no longer the default route for new prospects. Treat warm-up as a narrow gate, not a work queue.

Use a warm-up comment only when all are true:
- The prospect is high-fit T1/T2, not generic backlog.
- The buyer or company has recent relevant LinkedIn activity.
- The comment can be made naturally in under a few minutes.
- There is a concrete M1 or follow-up plan within 24-48h.

Do not create or preserve bulk “Warm-up comment:” tasks for JT. If a warm-up task is stale, channel-unclear, or not tied to an immediate follow-up, archive it or merge it into a buyer-channel validation task. Default outreach priority is proof-led referrals/warm intros, service-page/citation outreach, and review-only packets with a clear send path.

## Prospect Contact Completeness Rule
Outbound v2 (adopted 2026-07-02) replaces the old LinkedIn-plus-email requirement.

Send-ready requires:
- Named buyer: principal, owner, director of ops, or equivalent.
- Reachable channel: verified email OR accepted LinkedIn connection. Both are useful but no longer mandatory.
- Niche proof asset exists for the prospect's lane.

Do not treat unaccepted LinkedIn-only prospects as send-ready. Use them for connection/request path only. Track the channel used, delivered/bounced state, connection accepted state, artifact view state, and reply state before judging copy.

## Prospect Tier Action Rule
Tier prospects by binary gates before creating outreach tasks:

1. Live proof asset for the niche.
2. Reachable channel: verified email OR accepted LinkedIn connection.
3. Named buyer.
4. Trigger bonus: ops job posting, portfolio growth, PM software signal, stabilized/rent-regulated exposure, or similar.

- **T1:** all four gates. Use proof-led outreach and JT review. No custom demo/build/deck pre-reply.
- **T2:** gates 1-3. Use niche template, one personalized line, JT review, and strict measurement.
- **Hold/dead:** anything less. Do not load into send queue.
