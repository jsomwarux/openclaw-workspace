# Outreach Rules — Full Detail
> Referenced from AGENTS.md. Read this file when handling outreach tracking.

## Outreach Status Tracking Rule (mandatory)
When JT says he sent an outreach message, connection request, or follow-up to ANY prospect — do ALL of the following in the same turn, immediately. Do NOT defer.

**Step 1 — Update files:**
- Mark outreach-draft.md status line with what was sent, the date, and the next window
- Update the shortlist's contact line with current M-status and next action

**Step 2 — Create follow-up MC task (mandatory):**
Use the M-sequence timing below to calculate the follow-up window and push a HIGH task to MC immediately.
- **After M1 sent:** Create task "M2: [Company] — [Contact]" | sortOrder 50-90 | due in 3-4 days | description includes connection request note instruction + Drive draft link
- **After M2 sent (connection request accepted or pending):** Create task "M3: [Company] — [Contact]" | sortOrder 50-90 | due in 4-5 days | description includes free DM with soft CTA + Calendly link instruction
- **After M3 sent:** Create task "M4 (final touch): [Company] — [Contact]" | sortOrder 100+ | due in 7 days | description notes this is the last touch, pivot to email channel if no reply

**M-sequence timing reference:**
- M1 → M2: Day 3-4 (connection request note)
- M2 → M3: Day 4-5 after M2 (free DM once connected, soft CTA + Calendly)
- M3 → M4: Day 7 after M3 (final check-in, pivot to email)
- After M4 with no reply: move to cold — requeue in 90 days or switch channel

**Step 3 — Check for existing duplicate task before creating** (MC task dedup check mandatory).

- The MC task board tracks TO-DOs. The pipeline files are the source of HISTORY. Both must be updated.
- If JT marks a task Done on Mission Control without telling Eve, Eve will NOT automatically know to update the pipeline files. JT should still notify directly ("sent M2 to X") for accurate tracking.
