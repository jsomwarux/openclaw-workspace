### ContractorProof Packet [M5-TrustLayer] [M9-Vision] [M10-AgentNative]
**Concept:** A $19 contractor-risk packet for homeowners before they sign a remodel, roofing, HVAC, or electrical job. User enters contractor name/license number and uploads the estimate, COI, or text thread; the tool checks license age/status, insurance/bond proof, public complaints, permit responsibility language, payment schedule risk, lien-release gaps, and produces a hire/no-hire checklist.
**Lateral connection:** Contractor license lookup + fake-review detection + document OCR/vision + owner-friendly due diligence.
**levelsio equivalent:** More like a one-shot PhotoAI/InteriorAI paid output than a broad SaaS: one narrow artifact delivered instantly.
**JT stack fit:** OpenClaw cron can refresh state license lookup links, Claude can classify estimate/COI language, vision/OCR can extract PDFs/photos, and Next.js can ship a thin upload/report flow. JT's property/construction workflow knowledge prevents this from becoming a generic "contractor reviews" site.
**Revenue model:** $19 per packet, $49 for 3-packet comparison, optional $9/mo watchlist for permit/license/insurance changes during a project.
**Longevity signal:** Homeowner contractor risk is secular; licensing, insurance, deposits, change orders, and lien releases will stay confusing even as review sites improve.
**Research signal:** Weekly trends had a verification cluster around contractor/license checks. Brave surfaced Reddit threads asking where to verify contractor licenses and how to find reliable contractors, with users recommending license, insurance, bond, references, written scope, permits, milestone payments, and lien releases. Source examples: `reddit.com/r/HomeImprovement/comments/1rultbp/...`, `reddit.com/r/AskLosAngeles/comments/1odn95m/...`.
**Behavioral demand:** Fear of loss and embarrassment avoidance. A homeowner is about to wire thousands of dollars and wants certainty before getting burned; the packet gives a concrete reason to pause or negotiate now.
**TrustMRR pattern:** Adjacent to API validation/infrastructure and local-business trust tools like 1Lookup and LocalRank, but not a copycat. This validates the "paid verification artifact" pattern, not the exact vertical.
**Agent-native angle:** A real estate/property-management agent could buy this mid-task while vetting vendors for an owner. Output: JSON risk fields, source links, PDF checklist, and next approved questions to ask the contractor. Price: $19/report.
**TikTok Shop angle:** None. This is not social commerce.
**Creativity check:** Would not appear in a generic passive-income list. Generic "contractor directory" would be saturated; the packetized due-diligence artifact is the wedge.

**levelsio equivalent:** A paid micro-tool like PhotoAI, but for "turn this product claim into safer, evidence-backed copy."
**JT stack fit:** Glow Index already has ranking/evidence posture, skincare niche intel, and safe-claim constraints. Claude can rewrite and classify claims; OpenClaw can monitor trend ingredients from Exploding Topics; n8n can create weekly alert emails.
**Revenue model:** $15 one-off card, $29/mo for 20 cards, $99/mo microbrand plan with weekly trend-risk brief.
**Longevity signal:** Beauty claims, affiliate marketing, and social commerce will keep colliding with compliance and buyer skepticism. Even if platforms change, sellers still need safer copy and shoppers still need evidence.
**Research signal:** Exploding Topics fresh file shows beauty/personal-care growth around milky moisturizer, glass skin cream, repair serum, firming neck serum, centella serum, cica cream, builder gel, and adjacent trend ingredients. Web search surfaced AI product-claims checking tools and 2026 advertising-law content emphasizing objective claim substantiation. X search surfaced high-engagement AI-generated TikTok Shop selling examples, which confirms the commerce/content pressure but also raises compliance risk.

**Creativity check:** Generic "AI copywriter for ecommerce" is dead. Beauty claim-risk cards with evidence scoring and affiliate-safe language is specific enough to keep.
