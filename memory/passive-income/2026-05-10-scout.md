## 6 Raw Ideas

### 1. PDRN Decoder [M6-ExplodingTopics]
**Concept:** A narrowly focused skincare evidence/ranking site for the PDRN/salmon-DNA boom: rank PDRN serums, creams, toners, masks, ampoules, and clinic procedures by ingredient quality, claim risk, skin-type fit, and price-per-active. Audience: skincare buyers who have heard the trend but do not know which format is legitimate vs. TikTok bait.
**Lateral connection:** Glow Index-style ranking engine + fast-moving K-beauty ingredient craze + OpenFDA/ingredient risk lookup.
**levelsio equivalent:** PhotoAI/InteriorAI pattern, but the transformation is “turn noisy beauty trend into a ranked buying decision.” Also a mini Glow Index vertical, not a full skincare rebuild.
**JT stack fit:** Reuse the Glow Index ranking architecture and ingestion habits. Claude classifies INCI lists, claim types, irritation risk, product format, and user skin goals. n8n watches new PDRN products weekly. Next.js publishes SEO pages like “best PDRN serum for sensitive skin.”
**Revenue model:** Affiliate links to retailers + $9 one-time “routine fit report” generated from the user’s current shelf/photo + later brand-sponsored listing disclosures.
**Longevity signal:** The exact ingredient may peak, but the pattern repeats every year: snail mucin, exosomes, hypochlorous acid, PDRN, methylene blue serum. Build as `Ingredient Decoder` with PDRN as the first vertical.
**Research signal:** Exploding Topics shows multiple PDRN terms at 99x+ growth: PDRN serum 60,500 monthly searches, PDRN cream 40,500, PDRN mask 6,600, PDRN stick balm 4,400, salmon DNA microneedling 3,600. CPCs are low, meaning SEO is still early.
**Creativity check:** A “skincare blog” is generic. A trend-specific ingredient decoder/ranking engine for PDRN formats is not a top-10 passive-income idea. ✅
**Agent-purchasable angle:** Yes. A beauty-shopping agent could buy a $3 JSON “ingredient claim risk + best-fit product” result while assembling a routine for a user with sensitive skin. Output: JSON + markdown recommendation.

### 2. RouteShade [M7-GTrends] [M8-API]
**Concept:** A route planner for runners who care about conditions that generic maps ignore: shade, wind, pollen, heat index, hills, bathroom/water stops, safety, and stroller/dog friendliness. It ranks routes by “comfort score” for today’s exact weather window.
**Lateral connection:** Google Trends breakout for running route planner + Open-Meteo weather + Strava route culture + local safety/amenity scraping.
**levelsio equivalent:** Nomads.com directory logic applied to micro-routes: rank places by a specific lifestyle job-to-be-done, then compound via SEO and user-submitted routes.
**JT stack fit:** Open-Meteo gives weather, Strava API can enrich activity/segment context, Claude summarizes route tradeoffs, and a simple Next.js map UI can launch with static city pages before full routing. OpenClaw can refresh “best run right now” pages every morning.
**Revenue model:** $6/mo city-specific premium route packs; affiliate to running gear and hydration products; $19 one-time race-week route planner for travelers.
**Longevity signal:** Running does not go away, cities keep getting hotter, and people increasingly plan around climate comfort. “Route with shade at 5PM in July” is more durable than generic run tracking.
**Research signal:** Google Trends: `running route planner` +2606.7% over 30 days. Exploding Topics also shows smart insoles and long-range eBikes rising, pointing to outdoor mobility optimization demand.
**Creativity check:** “Make a fitness app” is generic. “Weather-aware shade/safety route scoring for runners” is a specific utility with a data flywheel. ✅
**Agent-purchasable angle:** Yes. A travel or fitness agent could buy a $5 route pack while planning a user’s trip or marathon taper week. Output: GPX/KML + route score JSON + map link.

### 3. CollectionProof [M7-GTrends] [M9-Vision]
**Concept:** A photo-first collection organizer for niche collectors — vinyl, board games, anime figures, Pokémon cards, sneakers, watches — that identifies items from photos, normalizes names/editions, flags duplicates, estimates replacement value, and exports insurance-ready inventory.
**Lateral connection:** Google Trends breakout for `collection organizer` + AI vision/OCR + existing rankable data APIs like Discogs, BoardGameGeek, RAWG, Jikan, Open Library, and TMDB.
**levelsio equivalent:** New pattern: “PhotoAI for personal asset catalogs.” The paid transformation is turning a messy shelf/box into a structured collection database.
**JT stack fit:** Vision model extracts title/edition/condition from shelf photos and close-ups. Claude resolves messy matches. API library supplies structured metadata by vertical. n8n can run periodic value refreshes. Next.js delivers shareable collection pages and insurance PDF export.
**Revenue model:** $12/mo for value tracking + $19 one-time “inventory my shelf” batch scan; affiliate to storage, grading, sleeves, and insurance partners.
**Longevity signal:** Collecting is secular and value tracking gets more important as collections get expensive. The user’s catalog becomes sticky once built.
**Research signal:** Google Trends: `collection organizer` +44.3%. Web result for “rate my home gym setup” also showed strong photo-audit behavior; users already post photos for judgment. The gap is turning photos into structured, reusable records instead of comment threads.
**Creativity check:** “Sell collectibles” is listicle fodder. “Vision-powered collection inventory + value/risk refresh for specific collector verticals” is not. ✅
**Agent-purchasable angle:** Strong. A home-insurance, estate-planning, or resale agent could buy a $15 inventory enrichment pass from user photos. Output: CSV + JSON + PDF inventory.

## levelsio Portfolio Gap Observations
- The best gaps this week are not broad apps. They are “specific decision engines” where the user already has purchase intent but the web is full of generic affiliate sludge: PDRN products, air purifiers, perfume, smart fitness gear.
- A repeatable pattern is emerging: combine one fast-growing consumer trend with one hard scoring layer. Examples: CADR room fit, ingredient claim risk, route weather comfort, collection value normalization. This is more defensible than yet another AI chatbot.
- Agent-purchasable microservices only make sense where the output is structured and immediately actionable. Route packs, procurement shortlists, inventory exports, and gift recommendations fit. Broad “agent marketplace” ideas still feel premature.
