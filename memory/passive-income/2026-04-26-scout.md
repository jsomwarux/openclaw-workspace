## 6 Raw Ideas

### 1. ROUTESAFE
**Concept:** AI route planner for runners who care about safety, lighting, elevation, bathrooms/water stops, and “not getting lost mid-workout.” User enters distance + starting point + vibe (“quiet 5K after work,” “well-lit long run,” “hill repeats”) and gets route options scored by safety/friction, not just distance.
**Lateral connection:** Google Trends “running route planner” +3223.3% + review signal from Footpath users saying they need a better niche solution for avoiding getting lost during workouts.
**levelsio equivalent:** Nomad List-style city scoring, but for running routes: lightweight directory + scorecards + community-submitted route data.
**JT stack fit:** Next.js route pages, OpenClaw scheduled refresh, n8n for route ingestion, map APIs, automated X/Reddit research for local runner complaints. No mobile app required for MVP: web-first route generator + shareable route pages.
**Revenue model:** $5/mo premium for saved routes, GPX exports, and weather-aware route adjustments; affiliate links for running shoes/gear; local run-club sponsorship pages.
**Longevity signal:** Running is secular and local. Existing route tools optimize maps, not runner-specific anxiety: safety, bathrooms, lights, surface, hills, and “don’t make me think.”
**Research signal:** Google Trends: running route planner +3223.3% current 32.2; sports affiliate programs up to ~20% commissions; web search surfaced running app complaints around needing better navigation mid-workout.
**Creativity check:** Would NOT appear in a “top 10 passive income” list ✅

**Lateral connection:** Beauty subscription box +1693.9% GTrends + affiliate comparison model + AI personalization. Not another beauty box; a decision layer over the fragmented market.

**Longevity signal:** Beauty subscriptions churn because people receive irrelevant products. A matcher that reduces regret has recurring value whenever boxes, prices, and brand lineups change.

### 3. STAINWISE
**Concept:** AI stain-removal decision tool. User selects or uploads stain type + fabric + how long it has been sitting; tool gives a ranked treatment plan and warns what NOT to do before the stain sets permanently.
**Lateral connection:** X finance/household signal: “LLMs are bad at finance… there is a right answer for ‘how do I get this stain off my blouse’” + consumer need for narrow, reliable, non-chat AI.
**levelsio equivalent:** Tiny utility site like PhotoAI in reverse: one focused input, one useful answer, monetized through affiliate products.
**JT stack fit:** Next.js utility pages, OpenRouter structured recommendations, n8n content pipeline for SEO pages (“red wine on linen,” “oil on cotton,” etc.), affiliate links to stain removers/laundry products.
**Revenue model:** Affiliate commissions on detergents, enzyme cleaners, stain pens, fabric care products; display ads on SEO landing pages; $4.99 printable laundry cheat sheet bundle.
**Longevity signal:** Stains are evergreen, urgent, and high-intent. People search at the moment of need and will click product links if the plan is credible.
**Research signal:** X: “there is a right answer for… ‘how do i get this stain off my blouse’”; web search found CleanBot AI/Stain Solution exists, which validates demand but the SEO/content lane still looks underbuilt.

## levelsio Portfolio Gap Observations

1. **The best current opportunities are decision layers, not new communities.** “Which route should I run?”, “which beauty box fits me?”, “which game should I platinum?”, “which dog food is safe enough?” are all small recurring decisions with affiliate paths.

2. **Trend-ingredient pages are the fastest SEO arbitrage.** PDRN, methylene blue serum, kojic/turmeric soap, beef tallow, and hypochlorous acid all create the same pattern: consumers hear the term before trustworthy comparison infrastructure exists.

3. **Fitness route planning looks more promising than another workout tracker.** “Gym workout tracker” is huge but saturated. “Running route planner” is almost as hot and has clearer unsolved friction around safety, bathrooms, lighting, terrain, and getting lost.

4. **AI works best here as constrained scoring.** Avoid open-ended chatbot products. The viable properties return ranked tables, checklists, safe/avoid labels, route scores, and personalized quiz matches.

5. **API signals were weak this week.** `weekly-apis.md` had no new APIs and only generic public-API directory chatter, so this week’s strongest ideas come from trend momentum + consumer comparison gaps rather than fresh API arbitrage.
