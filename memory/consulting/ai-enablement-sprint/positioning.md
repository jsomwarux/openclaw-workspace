## 7-day deliverables
1. Workflow map: intake → handoff → exception → approval → update
2. AI-fit opportunity list: what to automate, augment, or leave human-owned
3. Pilot governance checklist: risk, approval, data, owner, escalation
4. Tool recommendation: n8n, Agentforce, internal GPT, or no-code stack as appropriate
5. Measurement plan: time saved, backlog reduced, response time, error rate, adoption
6. Rollout checklist: who owns it, who uses it, what changes first

## Architecture Pattern to Reuse — Supervisor / Specialist / Judge / Human

Use this when explaining practical AI implementation to buyers:
- **Supervisor:** routes the request and decides which workflow path applies.
- **Specialists:** retrieve documents, check structured data, run analysis, draft outputs, or update systems.
- **Judge:** checks confidence, completeness, source support, and risk before action.
- **Human:** owns approvals, exceptions, overrides, and accountability.

Buyer translation: the goal is not a chatbot. The goal is an operating workflow where AI does narrow work, the system checks itself, and the right person reviews the right exception.

This strengthens the AI Enablement Sprint and AgentGuard positioning: every serious AI workflow needs a judge/handoff layer, not just a generator.
