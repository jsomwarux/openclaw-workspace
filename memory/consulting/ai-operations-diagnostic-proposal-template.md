# AI Operations Diagnostic — Proposal Template

## Subject / Title
AI Operations Diagnostic for [Company]

## Opening
[Company] is at the point where AI could create leverage, but the highest-value first build should come from the workflow, not from a generic tool decision.

This diagnostic maps how the work actually moves today, identifies the highest-ROI bottlenecks, and scopes the first production system to build.

## Scope
Over 2 weeks, I will review one priority operating area and produce a practical implementation roadmap.

Best-fit workflow examples:
- Back-office reporting
- Property/family-office operations
- Construction status and owner updates
- Vendor/document tracking
- Finance/approval workflows
- CRM or intake routing
- Recurring research/data workflows

## Deliverables
1. Current-state workflow map
2. Bottleneck and exception inventory
3. Ranked automation opportunities, usually 3–5 use cases
4. Recommendation on what to build first, defer, or avoid
5. Human-in-the-loop and risk design
6. First-build scope with timeline, dependencies, acceptance criteria, and implementation estimate

## Process
### Week 1 — Discovery + workflow map
- Stakeholder interviews
- Systems/data review
- Current process mapping
- Exception and handoff capture

### Week 2 — Ranking + first-build scope
- Use-case scoring
- Build/no-build recommendation
- Roadmap
- First implementation proposal

## Pricing Frame
Diagnostic fee: $3,500–$7,500 depending on workflow complexity, stakeholder count, systems access, and deliverable depth.

Optional: credit part of the diagnostic fee toward the first implementation if the build is approved within [X] days.

## Why this before implementation
The diagnostic prevents wasted spend on tools, chatbots, or broad AI initiatives before the company knows which workflow deserves automation first.

## Next Step
If this makes sense, the next step is a 30-minute scoping call to choose the workflow area and confirm access requirements.
