# AI Operations Diagnostic — Proposal Template

Created: 2026-05-05
Use for warm leads before pitching a large implementation build. Adapt for Marketsmith, family offices, property operators, construction teams, hospitality, or back-office ops teams.

## Client-Facing Title
AI Operations Diagnostic: identify the highest-ROI AI/automation system before committing to a larger build.

## Positioning
Most businesses do not need to start with a chatbot, a full-time AI hire, or a new SaaS platform.

They need to know which workflow is worth automating first, what data/systems are involved, where human approval is required, and what a safe production build should look like.

## Best-Fit Buyer
This is best for teams where important work moves through email, spreadsheets, portals, local files, finance systems, CRM, property systems, or manual follow-up.

Common fits:
- Family offices / property operators
- Construction and real estate teams
- Finance and back-office operations
- Hospitality and service businesses
- Product teams evaluating AI-enabled workflow/product initiatives

## Scope
### 1. Workflow Discovery
Map one high-priority workflow from trigger to completion:
- systems involved
- people/owners
- handoffs
- approvals
- manual copy/paste points
- exception states
- sensitive data boundaries

### 2. Exception Inventory
Identify what gets stuck, missed, duplicated, delayed, escalated, or hidden from leadership.

### 3. Automation Opportunity Ranking
Score 3–5 use cases by:
- frequency
- time drain
- business impact
- data readiness
- automation risk
- implementation complexity
- proof/referral value

### 4. First-Build Recommendation
Define the first system worth building, including:
- acceptance criteria
- human-in-the-loop rules
- inputs/outputs
- integrations
- audit trail/logging needs
- risks/dependencies
- estimated timeline and price range

## Deliverables
- Current-state workflow map
- Bottleneck / exception list
- Ranked use-case matrix
- Recommended first build
- Human approval and risk design
- Implementation roadmap
- Optional implementation proposal

## Timeline
Default: 2 weeks from kickoff.

## Pricing Frame
Suggested range: $3,500–$7,500 depending on workflow complexity, stakeholder count, data access, and whether technical validation/prototyping is included.

Optional commercial term: diagnostic fee can be credited toward implementation if the client proceeds within 30 days.

## Implementation Upsell Frame
If the diagnostic shows a clear ROI case, the next step is a contained production build, not an open-ended transformation project.

Example first builds:
- exception dashboard
- document/intake classifier
- local-first reporting workflow
- vendor/document tracker
- owner-ready weekly brief
- CRM/Salesforce AI agent
- n8n/Python workflow automation with audit trail

## CTA Language
> Worth starting with a short diagnostic so we can identify the highest-ROI system before committing to a larger build?

## Marketsmith Adaptation
Frame as: Product/AI diagnostic for MSIGenerate or new initiatives.

Emphasize:
- clarify product workflow/user problem
- identify where AI improves an existing process
- define a buildable first version
- avoid vague innovation work
- deliverables over hours

## Family Office / Altmark Referral Adaptation
Frame as: Local-first automation diagnostic for family-office back-office workflows.

Emphasize:
- QuickBooks/Desktop-style environments
- local files and sensitive docs
- property/finance workflows
- audit trail
- human approval for financial/external actions
- no generic SaaS replacement

## Construction Adaptation
Frame as: AI implementation diagnostic for field-to-office workflows.

Emphasize:
- field notes/photos → punch items/customer updates
- change order and approval tracking
- vendor/subcontractor follow-up
- owner-visible exception dashboard
- risk reduction and cycle-time improvement
