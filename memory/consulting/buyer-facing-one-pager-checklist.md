# Buyer-Facing One-Pager Checklist

Use this before sending or uploading any JT Consulting one-pager/deck insert.

## Required fields

- [ ] Offer name is specific, not “AI consulting.”
- [ ] Buyer is named by role and operating context.
- [ ] Pain is phrased in the buyer's workflow language.
- [ ] Scope says exactly what JT inspects/builds.
- [ ] Output is concrete: map, exception inventory, dashboard, tracker, runbook, first-build scope, approval rules, logs.
- [ ] Timeline/range is included only if current and defensible.
- [ ] CTA asks for one low-friction next step.
- [ ] Proof is anonymized or permissioned.
- [ ] Claims avoid unsupported ROI, time saved, named clients, or confidential workflow details.
- [ ] Human approval/audit trail boundaries are explicit.
- [ ] Source file and Drive location are noted.
- [ ] Hidden/public status is noted if the asset maps to a website page.

## Fast verdict

- **SHIP:** specific buyer, safe proof, clear CTA, no unsupported claims.
- **FIX THEN SHIP:** useful but missing one of buyer/scope/proof/CTA/source status.
- **HOLD:** relies on unpermissioned proof, stale strategy, vague AI language, or an offer lane JT is not actively pursuing.
