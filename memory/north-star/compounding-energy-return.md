# Compounding Energy Return (CER)

Created: 2026-05-15

## Source Signal
Screenshot of Claude Opus proposing **Compounding Energy Return**: a metric for how much usable physical, cognitive, emotional, and relational energy a decision/hour/commitment returns over time relative to its upfront cost.

## JT Interpretation
Use CER as a qualitative decision filter, not a fake scientific score.

The useful question:
> Does this action give JT more future energy, leverage, clarity, money, relationships, or optionality than it costs?

## Why It Matters
JT's North Star is financial freedom and control over time. The highest-leverage work should not only produce output today; it should reduce future drag or increase future capacity.

High-CER work compounds:
- client systems that become reusable IP
- proof assets that improve sales trust
- automations that reduce repeated decisions
- health/sleep actions that preserve earning capacity
- relationships that create warm distribution
- content that sharpens positioning
- runbooks/checklists that prevent repeated mistakes

Low-CER work drains:
- polishing assets nobody sees
- adding features before distribution
- context-switching across too many projects
- reviewing low-quality agent output repeatedly
- generic research with no decision attached
- speculative builds with no buyer, trigger, or distribution path

## Decision Filter
Before adding or prioritizing optional work, ask:
1. **Energy return:** Will this make future work easier, clearer, or more valuable?
2. **Compounding path:** Does it create an asset, relationship, system, proof point, or reusable pattern?
3. **Time horizon:** Does the return show up this week, this month, or only in a vague future?
4. **Drag cost:** Will it create maintenance, context switching, obligations, or fragile complexity?
5. **North Star fit:** Does it support consulting revenue, app distribution, crypto edge, or health?

## Routing Rules
- High CER + near-term revenue/proof: prioritize.
- High CER + long-term optionality: schedule only after cash-critical tasks.
- Low CER + urgent obligation: do minimum viable version.
- Low CER + optional: defer or kill.
- High excitement but unclear CER: convert to `memory/future-signals.md` with a trigger.

## Integration Points
- Mission Control task reviews: prefer tasks with compounding assets over one-off busywork.
- Weekly North Star review: flag one high-CER action and one low-CER task to kill/defer.
- Health layer: low sleep/energy lowers available CER; protect recovery before optional work.
- Consulting: favor deliverables that become reusable IP, proof, templates, or referral assets.

## Anti-Pattern
Do not turn CER into a spreadsheet with fake precision. A simple High / Medium / Low judgment is enough.
