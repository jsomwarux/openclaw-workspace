# North Star Review Rubric

Purpose: keep JT moving toward financial freedom + control over time, not just accumulating work.

## North Star
Financial freedom and control over time through multiple high-earning, low-maintenance income streams, ideally managed by specialized AI agents that only escalate urgent decisions.

## Priority Order
1. AI implementation consulting
2. App/product building + marketing toward passive or semi-passive income
3. Crypto market monitoring/opportunity scanning
4. Health as the base layer

## Income Thresholds
- Safe: ~$10K/month
- Free: ~$30K/month
- Rich: ~$100K/month

## Non-Negotiables
- Sleep
- Health
- NYC/location stability
- Do not force JT into hands-on developer positioning

## Weekly Lane Scores
Score each lane 1-5 on:
1. Near-term cash flow
2. Long-term leverage / passive-income potential
3. Distribution or audience growth
4. Credibility for AI implementation consulting or product-builder identity
5. Compounding Energy Return (5 = returns future energy/leverage/proof, 1 = draining maintenance/context-switching)
6. Strategic fit with North Star

Total possible: 30.

## Decision Rules
- 24-30: Double down this week
- 18-23: Maintain / focused next step only
- 12-17: Pause unless it unblocks a higher lane
- <12: Kill, archive, or explicitly defer

## Active Lanes to Review
- Consulting revenue pipeline
- Aya/client delivery
- Content/distribution engine
- Apps/products: Vista, Nash Satoshi, Glow Index, Action Arena, Dynasty Simulator
- Crypto monitoring/x402
- Job market/opportunities
- Health/energy
- Infrastructure/agent systems

## Revenue Command Center
Estimate monthly run-rate and path to thresholds every week.

Track by source:
- Consulting: closed revenue, active proposals, likely next deal, retainer potential
- Apps/products: current revenue, active users/audience, monetization path, next distribution move
- Crypto: income/opportunity state, asymmetric watchlist, risk level
- Job market: salary opportunity quality, fit with non-dev AI implementation identity
- Content-driven leads: audience growth, inbound signals, proof assets created

Required weekly fields:
- Current known monthly run-rate
- Gap to Safe ($10K/mo), Free ($30K/mo), Rich ($100K/mo)
- Highest-probability next dollar
- Fastest credible path to +$1K/mo
- Fastest credible path to +$10K/mo

## Project Overinvestment Guard
Every active project must pass at least one of these tests:
1. Near-term cash flow
2. Durable distribution/audience
3. Reusable product/IP
4. Consulting proof/credibility
5. Health/energy preservation
6. High CER: reduces future drag or creates reusable leverage

If a project passes none, recommend pause/kill/defer. If it passes only one weakly, cap the next action to a small validation step.

## Product Portfolio Review
For each app/product, assess:
- Target user
- Current proof/audience
- Monetization path
- Next distribution move
- Biggest risk
- Continue / pause / kill / validation-only

Default bias: do not build more features until a distribution or monetization bottleneck is named.

## Required Output
1. Executive read: 3-5 bullets
2. Revenue command center: run-rate, gaps, highest-probability next dollar
3. Consulting sales engine: pipeline health, follow-ups due, next-dollar action
4. Proof + distribution: proof assets found, missing distribution, next case study/post
5. Product growth calls: continue / pause / kill / validation-only for each product
6. Opportunity gate: promote / defer / prune decisions with trigger conditions
7. Lane scorecard table
8. Double down / maintain / pause / kill decisions
9. Project overinvestment warnings
10. Health/focus risk and workload adjustment
11. Top 3 actions for the week
12. CER call: one high-CER action to prioritize and one low-CER task/project to stop, defer, or minimum-viable
13. One configuration/system improvement Eve should make
14. Saved file path

## Action Ownership
Every recommended action must be labeled:
- Eve can do autonomously
- JT-only decision/send/review
- Blocked until external input

## Noise Control
Do not create new crons unless the work is time-sensitive, recurring, and cannot be folded into an existing review. Prefer state files + Mission Control tasks over extra recurring jobs.

## Guardrail
Do not create vague reflection. The review must produce decisions and next actions. If data is missing, state the missing data and make the best conservative call.
