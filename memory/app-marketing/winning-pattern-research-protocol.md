# App Marketing OS — Winning Pattern Research Protocol
Date: 2026-05-19
Owner: Eve

## Purpose
Find repeatable low-cost acquisition patterns from successful/similar apps, score them, and convert only the strongest patterns into measured experiments.

## Research Cadence
- **Daily light scan:** collect 3-7 candidate patterns from fresh sources when app-marketing work is active.
- **Weekly synthesis:** score candidates, select experiments, update Mission Control.
- **Monthly teardown:** deep competitor/channel teardown for each active app.

## Required Source Mix
Use at least 3 source types before promoting a pattern:
1. App Store / Google Play reviews and screenshots.
2. TikTok / Reels / Shorts examples.
3. X posts and replies.
4. Reddit/community threads.
5. YouTube videos/transcripts or creator breakdowns.
6. SEO results: autocomplete, “best X app,” “X alternative,” comparison pages.
7. Product Hunt / indie launch writeups.
8. Newsletters, roundup writers, app reviewers, niche blogs.

## Pattern Scoring
Score 1-5 each. Only patterns scoring **24+/35** become experiments.

| Criterion | Question |
|---|---|
| Audience fit | Is this JT's actual user? |
| Repeatability | Can JT/Eve do this weekly without huge manual load? |
| Proof | Is there visible engagement, search demand, backlinks, reviews, or adoption? |
| Cost | Can this run under $0-$200/month? |
| Conversion intent | Does the user want an app/tool now? |
| Shareability | Does it create an artifact users/creators want to share? |
| Competition gap | Are incumbents weak or generic here? |

## Promotion Rule
A pattern is not “accepted” until it becomes an experiment card with:
- source/proof
- hypothesis
- asset needed
- channel
- CTA
- tracking method
- success metric
- kill/iterate/scale rule

## Anti-Slop Rules
- Do not copy creators. Extract structure only.
- Do not treat one viral post as proof unless comments show real user intent.
- Do not promote generic `I built an app` formats unless niche evidence supports them.
- Do not use Product Hunt/directories as proof of durable users.
- If tracking is impossible, use the pattern only for durable assets like SEO pages or share-card specs.
