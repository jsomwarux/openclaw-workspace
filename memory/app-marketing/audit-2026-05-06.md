## Current Systems Reviewed

### Vibe Marketing Agent
Location: `agents/vibe-marketing/AGENT.md`

What it already does well:
- Generates product-specific weekly content for X, TikTok/ReelFarm, Reddit, and monthly LinkedIn.
- Separates product marketing from JT's personal brand content.

Weaknesses / gaps:

- Performance log is too thin to support real winner/loser learning: only 6 rows, all pending/zero metrics; queue has 119 entries but performance feedback is weak.
- State shows screenshot Drive folders missing for Nash Satoshi and Vista, but this may not matter if JT's laptop ReelFarm setup already has the screenshot assets. Treat as an asset-map question, not an automation blocker.
- Vibe Marketing does not own directory submissions, SEO landing-page backlog, App Store optimization, or cross-app weekly scoreboards.
- Existing Reddit/X generation should be tightened only after review; do not create a parallel Reddit/X generator.

## What Not To Do
- Do not create a second Reddit generator. Existing Reddit generation already exists and has guardrails.
- Do not create a second X generator. Existing product X + personal brand + @dynastyjig systems already exist.
- Do not use “I built X for Y” as Reddit default; many subreddits will treat that as self-promotion.

## Recommended System Architecture

### App Marketing OS = control layer, not replacement
New source of truth should live under `memory/app-marketing/` and coordinate existing systems.

Responsibilities:
1. App registry / stage / channel map.
2. Weekly scoreboard and analytics requirements.
3. Directory/backlink submission backlog.
4. SEO landing-page backlog.
5. ASO checklist for App Store apps.
6. Future-app onboarding template.
7. Promotion boundaries per platform.

Existing systems stay in place:
- Vibe Marketing = existing product content generator and legacy TikTok prompt/queue surface; JT's laptop owns actual ReelFarm execution.
- ReelFarm Intel = TikTok slideshow trend intelligence.
- Content Scheduler/Calendar = JT personal brand.
- Sports GM = @dynastyjig / Action Arena / Dynasty Simulator audience-building.

## Highest Priority Gaps
1. Measurement: performance-log metrics are too weak. Automation cannot optimize without actual views/clicks/signups/downloads.
2. Registry accuracy: Action Arena and Dynasty Simulator need correct marketing records; Glow Index status needs activation conditions.
3. Durable discovery: directory submissions + SEO pages are missing as first-class systems.
4. ASO: Vista and future App Store apps need keyword/screenshot/description optimization loop.
5. Asset map: confirm where Vista/Nash/Glow screenshots live in JT's laptop/ReelFarm setup; do not build a duplicate asset pipeline unless needed.
