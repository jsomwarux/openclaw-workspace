## Draft

A sales demo can make AI look ready before the operation is ready.

The gap shows up after the call:

- the intake lives in email
- the approval rule lives in one manager's head
- the customer record is half-updated
- the escalation path changes by account
- the handoff after the agent draft has no owner

This is why AI consulting keeps getting bundled with SOWs, cloud architecture, implementation planning, and value ownership.

A useful demo survives Monday morning because the operating model is already mapped:

- which source is trusted
- which action is policy-gated
- what gets written back to the system of record
- who approves expensive or sensitive cases
- what metric proves the workflow improved

Agents become safer when the surrounding workflow has source ownership, approval rules, writeback boundaries, and a value metric.

That is the work before implementation.
