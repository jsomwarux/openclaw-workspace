## LinkedIn Draft

A support inbox, CRM record, approval queue, and KPI dashboard should not become four separate AI projects.

That is where most adoption plans get expensive.

The team buys a tool.
Someone connects it to one workflow.
The demo works.

Then the real questions show up:

- Which customer context is trusted?
- Which action needs human approval?
- Who owns exceptions?
- What gets written back to the system of record?
- How do managers know the workflow is saving time or reducing risk?

The useful work happens before the agent acts.

Map the frontline workflow.
Name the inputs.
Separate clean actions from review-only actions.
Set the handoff rule.
Log the result.
Measure the outcome.

That is the operating model.

The agent is only one worker inside it.
