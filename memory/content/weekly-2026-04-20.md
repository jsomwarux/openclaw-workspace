## Friday — Tactical Insight: Workflow Defaults
**Pillar:** Tactical Insight | **Day:** Friday 2026-04-24

The design decision that changes every workflow: parallel or sequential.

Most automations run steps in sequence because that's how humans think about processes.

But many operational workflows fail because steps run simultaneously and create conflicts the system doesn't surface until later.

The fix isn't more logic. It's choosing the right default.

Parallel when order doesn't matter. Sequential when it does.

That single decision shows up in every build that runs correctly versus one that quietly breaks in production.

## Sunday — Behind the Build: AgentGuard Lesson
**Pillar:** Behind the Build | **Day:** Sunday 2026-04-26

First version of AgentGuard blocked everything that looked anomalous.

That approach generates more work than it prevents. Operations teams don't need a governance system that trips on legitimate edge cases.

The fix was counterintuitive: make the system report on anomalies without stopping them. Let the operator see what the model flagged and decide.

The governance layer works better when it's advisory, not enforcement.

That's not a security risk. That's designing for how the operations team actually works.
