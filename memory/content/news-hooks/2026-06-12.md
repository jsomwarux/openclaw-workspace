
## 09:31 - Agentic Pipelines now supports coding agents in CI
**Status:** Ready to post
**Hook:** Coding agents are moving into CI, so the real implementation question becomes approval gates, logs, and rollback rules.
**Source:** https://www.atlassian.com/blog/bitbucket/agentic-pipelines-now-supports-claude-code
**X Thread:**
A dev-platform vendor just moved coding agents into CI. Your automation can now assign a pipeline step to a coding agent and keep the work inside the build path.
---
The update lets teams choose a coding-agent provider for pipeline steps. That shifts the control point to repo permissions, approval gates, logs, and rollback rules.
---
For SMB ops, your agent needs a source, queue, approval owner, log, and rollback path before it touches real work. The coding stack is making that boundary normal. (via @Atlassian)
