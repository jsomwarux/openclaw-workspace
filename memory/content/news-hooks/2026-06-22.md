A new agent-security report says a malicious web page shown to a browsing agent can reach local MCP services and run processes on the host. That changes the risk model for office automations.
---
The mechanism matters: the agent renders the page, the page talks to local MCP, and local tools may expose file or shell access. The browser becomes a bridge into your workflow stack.
---
If your team uses agents with local tools, your team should treat web browsing like an approval boundary. Give the agent a narrow job, narrow tools, and a log you can inspect. (via @aiagentstore)
