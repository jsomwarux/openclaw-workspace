## Relevance Ladder

Use this order when choosing what to generate:

1. **Fresh trend + JT has earned angle** — best default. Recent platform-native signal from JT's niche, translated through JT's proof/taste.
2. **Current effort + useful lesson** — strong when there is enough concrete detail, especially Altmark, AI Operations Diagnostic, family-office/property ops, Marketsmith, app distribution, or buyer-channel enrichment.
3. **Fresh proof point/build** — strong when the work is complete enough to support a claim.
4. **Older proof tied to current motion** — acceptable only when it teaches something useful now.
5. **Evergreen operator truth** — allowed sparingly when it is sharp, non-generic, and not a repeat structure.

Current efforts do **not** have to anchor every post. They should prevent drift, not force weak content. If a fresh niche trend is stronger than current-effort material, use the trend. If neither a trend nor current effort supports a slot, skip or bank the slot rather than padding.

## Hard content priorities

1. Favor fresh niche/platform signals over stale internal proof.
2. Favor current efforts over old proof when both are viable.
3. No generic “AI agents are the future” posts.
4. Consulting posts should usually map to one of: diagnostic, exception visibility, local-first workflow, buyer-channel clarity, family-office/property ops, implementation judgment, or a current high-performing niche conversation.
5. App/product posts should usually map to one of: distribution loop, user feedback, rankings engine, app marketing system, AI-search/SEO surface, or a current high-performing indie/app marketing conversation.
6. Skip a post rather than filling a slot with stale content.
