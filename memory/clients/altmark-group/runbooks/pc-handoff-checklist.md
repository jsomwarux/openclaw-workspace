# Altmark PC Handoff Checklist

## Goal
Deliver the dedicated workflow PC to Altmark with enough documentation that Altmark understands what is running, how to access it, and how to verify the first workflow.

## Before Delivery
- Confirm delivery date/time with Yair/Navid.
- Confirm office/network access requirements.
- Verify PC boots cleanly.
- Verify required accounts/apps are logged in or documented.
- Verify automation folder structure exists.
- Verify backup/export location.
- Verify credential storage approach.
- Verify insurance expiration workflow test run.
- Capture screenshots for proof/runbook.

## At Delivery
- Walk Navid through machine purpose.
- Show folder structure and workflow files.
- Show logs / audit trail.
- Run one test workflow or dry run.
- Confirm what Altmark owns and where documentation lives.
- Confirm support path if machine/network/app breaks.

## Acceptance Criteria
- PC installed and accessible on-site.
- Insurance expiration workflow demonstrated or verified.
- Yair/Navid confirm they understand what is live.
- Open issues are logged with owner/date.
- Next workflow deposit/status is confirmed.

## Post-Delivery
- Update Client OS dashboard.
- Add screenshots to proof-assets folder.
- Send short weekly update.
- Ask for acceptance confirmation before packaging proof asset.
