## Event Scouting Checklist

For every researched event, capture:
- **Event / organizer:**
- **Date / venue:** exact, not “monthly”
- **Link:**
- **ICP relevance:** construction, property management, family office/property ops, wholesale distribution, skilled trades, Guyana/local-content, or SMB operators
- **Relationship value:** who JT could meet and why this room matters
- **Offer/proof readiness:** what JT can credibly say now without overclaiming
- **Next action:** RSVP / research / skip / create MC task
- **MC task:** title/id if created

## Monthly Scout Rules

- Run on first Monday only unless JT asks for an event scan.
- Buyer/operator rooms beat generic AI/networking rooms.
- Create a HIGH Mission Control task only when there is a real date, venue, clear ICP relevance, and a specific reason JT should attend.
- If no event is strong enough, log “no event found” with date and do not manufacture urgency.
- Do not use active-client names/proof in event positioning unless the proof gate allows it.
