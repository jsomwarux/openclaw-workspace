# Niche Intelligence Library

Purpose: agent-readable niche context that improves consulting teardowns, outreach, passive-income scoring, app marketing, and content. This is not an encyclopedic Obsidian vault. Every note must produce decisions, hooks, workflows, or build ideas.

## Operating Rule
A niche note is only useful if it changes at least one of:
- what JT builds
- who JT targets
- what JT says in outreach
- what content angle gets drafted
- what passive-income/app idea gets promoted or killed
- what AI Ops Teardown gets created

## Files
- `property-family-office.md` — property managers, family offices, finance-heavy real estate ops
- `construction.md` — construction field ops, contractor workflows, owner/PM visibility
- `wholesale-distribution.md` — inventory, order status, catalog knowledge, distributor workflows
- `skilled-trades.md` — trades businesses, scheduling, quoting, field/service workflows
- `skincare-glow.md` — Glow Index, skincare rankings, TikTok Shop/social commerce, pSEO/GEO
- `app-marketing.md` — mobile app growth, AppKittie, ASO, ReelFarm/TikTok, competitor intel
- `agent-native-apps.md` — agent-readable/passive-income apps, API/MCP/llms.txt products
- `sports-fantasy.md` — @dynastyjig, Action Arena, fantasy/betting product lanes

## Required Note Shape
Each niche file should maintain:
1. Buyer/user pains
2. Common workflows
3. Software/tools used
4. Automation opportunities
5. Objections/trust issues
6. Content angles
7. Outreach hooks
8. Proof assets/JT advantages
9. Passive-income/app ideas
10. Recent signals
11. Kill/defer rules
12. Next best action

## Update Rule
Update a niche file only when new research or client work changes a decision, offer, workflow, outreach hook, content angle, passive-income idea, or kill/defer rule. Do not append routine news or generic background. Keep entries concise and decision-oriented. If an update would not change an action, skip it.

## Agent Routing
- Consulting pipeline/outreach → read target niche file before drafting.
- AI Ops Teardowns → read relevant niche file before scoring/building teardown.
- Passive-income scout/strategist → read relevant niche files before promoting ideas.
- App Marketing OS → read `app-marketing.md`, `skincare-glow.md`, and relevant product niche file.
- Content generation → use niche files only when they improve specificity; do not force them into every post.
