# Agent-Native / Agent-Readable Apps

## Current Thesis
Build products humans value today and agents can discover, call, buy, and cite tomorrow. Avoid imaginary-agent vaporware.

## Buyer/User Pains
- Agents need reliable narrow outputs while completing tasks.
- Humans need provenance, approval boundaries, and clear artifacts.
- Most “for agents” products have no urgent job-to-be-done.

## Common Workflows
- Verification, scoring, comparison, lookup, enrichment, report generation.
- Rule-change monitoring.
- Data refinery from messy input to structured output.
- Vertical memory vaults and source-of-truth layers.

## Tools / Systems Used
APIs, MCP, OpenAPI, llms.txt, x402, Stripe/agent wallets, webhooks, JSON/CSV/PDF/markdown outputs.

## Automation Opportunities
- Agent-callable paid reports.
- Agent-purchasable API/workflow endpoints with pricing, receipts, and spend controls.
- Source-cited verification endpoints.
- Compliance/rule-change alerts.
- Spreadsheet-to-agent desks.

## Objections / Trust Issues
- Agentic commerce may be early.
- Need human buyer value today.
- Must be instant/low-support with clear pricing and schemas.

## Content Angles
- “Do not build apps for imaginary agents. Build outputs agents can use.”
- “The product is the artifact: JSON, report, citation, next action.”
- “x402 matters when it makes a useful endpoint agent-purchasable, not because crypto is the point.”
