## Afternoon Session — n8n Webhook Debugging (2–5 PM EST)

### Root Cause Chain (fully documented)
n8n v2 webhook issue had 4 layered problems, all resolved one by one:

**Problem 1 — Draft vs Published:** API-created workflows get `active=1` in DB but NO entry in `workflow_history` or `workflow_published_version`. n8n only registers production webhooks for "published" workflows. Fix: manually insert into `workflow_history` + `workflow_published_version` + set `workflow_entity.activeVersionId`.

**Problem 2 — activeVersionId mismatch:** n8n checks if `workflow_entity.activeVersionId` == `workflow_published_version.publishedVersionId`. These must match AND both must point to a valid `workflow_history.versionId`. Any mismatch = "0 published workflows" on startup.

**Problem 3 — webhook body double-encoding:** All 8 LLM nodes (S1/S2 × 4 models) had `specifyBody: "json"` but `jsonBody: "={{ JSON.stringify(...) }}"`. With `specifyBody: json`, n8n expects an object expression, not a pre-stringified string. Fixed to `={{ $('Build S1 Prompts').item.json.claudeBody }}`.

**Problem 4 — wrong Claude model:** `claude-3-5-sonnet-20241022` returns HTTP 404 on this Anthropic account. Tested multiple models. Only `claude-opus-4-0` and `claude-sonnet-4-0` return 200. Fixed to `claude-opus-4-0` in Build S1 Prompts + Build S2 Prompts.
