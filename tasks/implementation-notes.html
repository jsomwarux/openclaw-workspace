Content Swipe Optimization Fixes Notes

Status: Complete
Date: 2026-06-15
Task: Convert the @jts_14 swipe ledger from a prompt promise into a deterministic artifact contract
Top notes:
  1. The root issue is not missing research; it is missing deterministic packaging from research outputs into the required ledger.
  2. The generator should use local run artifacts only, with no network calls and no blind cron reruns.
  3. The content distribution guard should be able to fail closed on a missing ledger during recovery or generation checks.

Design Decisions

Add a local generator for memory/content/jts14-x-reference-ledger-YYYY-MM-DD.md instead of relying on a cron prompt to remember the ledger. Keep source classification conservative and inspectable.

Deviations

No live X or Notion fetch is part of this fix. The goal is to repair the artifact contract from already-saved local outputs and make future recovery deterministic.

Tradeoffs

The generator can infer lane and influence type from saved report text, but final judgment still remains visible in the ledger table. This is better than invisible prompt-state, and safer than inventing fresh signals.

Open Questions

Whether to patch the live cron prompt is still a separate systems-review decision because the repeated content cron failure can partially mutate state. The deterministic builder and guard hook are now available for safe recovery and verification.
